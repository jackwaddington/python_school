# School Simulator

A terminal-based Python simulation of a school, where almost everything is generated by a local LLM (Ollama). You set the parameters, the AI does the paperwork, and the simulation runs itself — producing weekly and monthly newsletters as files.

Built as a learning project for OOP in Python at Hive Helsinki.

---

## Setup

```bash
cd ~/github/python_school
source venv/bin/activate
ollama serve   # if not already running
python3 main.py
```

**Supported models:** `mistral`, `llama3.2`, `phi3`, `gemma2:2b`
Ollama defaults to `localhost:11434`.

---

## How It Works

1. You answer a few setup questions (numbers only — no AI needed yet)
2. You choose **auto** or **semi-auto** mode
3. The AI (or you) fills in the school's identity and content
4. The simulation runs for however many weeks you set
5. Weekly and monthly newsletters are written to disk

## Variables Reference

### Enforced manual — you must type these

| Variable | Question asked | What it controls |
|---|---|---|
| `seats` | How many seats does your school have? | School capacity |
| `num_courses` | How many courses will your school offer? | Number of courses created |
| `courses_to_graduate` | How many courses must a student complete to graduate? | Graduation threshold |
| `min_weeks` | Minimum weeks to complete a course | Lower bound of course duration |
| `max_weeks` | Maximum weeks to complete a course | Upper bound of course duration |
| `intake_interval` | How often will you take a new intake? (weeks) | Cohort frequency |
| `total_weeks` | How many weeks should the simulation run? | Length of the simulation |

### Your choice — type it or press Enter for AI (respects auto/semi mode)

| Variable | Question asked | What it controls |
|---|---|---|
| `subject` | What are you telling investors you teach? | School's field of study |
| `school_name` | Every great school needs a name. What's yours? | School name |
| Course names | What courses will you offer? | One name per course |
| Course material | Write the course material for '...' | Teaching content per course |
| Exam questions | Write 2 MCQ questions for '...' as JSON | Assessment per course |
| Student records | You need N student records... | First cohort population |

### Hardcoded — fixed in the simulation engine

| Variable | Value | What it controls |
|---|---|---|
| `student.ability` | random 0.3–0.9 | Probability of passing each weekly exam |
| Rough cohort ability | random 0.1–0.4 | Ability range when a cohort is flagged as rough |
| Rough cohort chance | 15% per new intake | Probability each new cohort is a disaster |
| Base dropout rate | 0.4% per week | ~55% cumulative dropout over 200 weeks |
| Strike dropout spike | 8% per week | Dropout rate during a teacher strike |
| Funding cut rate | 1% per week | Chance of losing 20% of seats that week |
| Teacher strike rate | 0.5% per week | Chance of a 3-week exam blackout starting |
| Strike duration | 3 weeks | How long a strike lasts |
| Students per seat | ×8 | First cohort size = `seats × 8` |
| Batch size | 10 students | Students generated per LLM call |
| Simulation start date | 2026-01-05 | Calendar date for week 1 |

| File | Class | Key attributes |

| `student.py` | `Student` | `ability`, `course_index`, `course_passes`, `graduated`, `dropped_out`, `cohort` |

| `newsletter.py` | `Newsletter` | `date`, `newsletter_type`, `recipients`, `content` |

| `llm_client.py` | `LLMClient` | `host`, `model`, `generate(prompt)` |
