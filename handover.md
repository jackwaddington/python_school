## 1. What This Project Is

A terminal-based Python simulation of a school, where almost everything is generated by a local LLM (Ollama/Mistral). The AI does all the boring work — inventing school names, course materials, student names with pronouns and allergies, exam questions — so you can focus on the OOP structure and simulation logic.

**There is intentionally no persistence.** Every run generates a completely fresh school. The AI is the database.

## 3. How It Works — The Flow

Run with: `python3 main.py` (activate venv first: `source venv/bin/activate`)

1. **LLM Setup** — ask for Ollama host (`localhost:11434`) and model name (`mistral`)
2. **School** — ask for school name, subject, seats. AI generates if user presses Enter
3. **Courses** — AI generates exactly 3 course names based on subject
4. **Material** — AI writes course material for each course (a few paragraphs)
5. **Exams** — AI generates 2 MCQ questions per course as JSON, stored in `course.exam`
6. **Students** — AI generates `seats × 8` students in batches of 10 as JSON — **CURRENTLY BROKEN**

The `ask_or_generate()` helper is used throughout — shows a question, lets user type, or calls the LLM if they press Enter.

## 4. The Bug To Fix First (Student Parsing)

Student batch generation fails with `json.JSONDecodeError`. Three root causes:

### Problem 1 — Mistral returns Python `None` instead of JSON `null`

JSON does not have `None`. It uses `null`. Fix: add these lines after `response = llm.generate(prompt)`:

```python
response = response.replace(": None", ": null")
response = response.replace(":None", ":null")
```

### Problem 2 — next_of_kin is a nested object

Mistral returns `next_of_kin` as a full nested object with name, relationship, phone, email. Just store the whole dict:

```python
student.next_of_kin = s["next_of_kin"]
```

### Problem 3 — Mistral sometimes wraps JSON in markdown backticks

Strip them before parsing:

```python
response = response.strip()
response = response.removeprefix("```json").removeprefix("```").removesuffix("```").strip()
```

### Full fixed student loop

```python
for i in range(batches):
    prompt = "Generate 10 fictional students as a JSON array. "
    prompt += "Each student needs: first_name, last_name, pronouns, email, allergies, next_of_kin. "
    prompt += f"Use {school.name.lower().replace(' ', '')}@school.com as email domain. "
    prompt += "For allergies use null if no allergy. Reply with ONLY the JSON array, no markdown."

    response = llm.generate(prompt)
    response = response.strip()
    response = response.removeprefix("```json").removeprefix("```").removesuffix("```").strip()
    response = response.replace(": None", ": null").replace(":None", ":null")

    try:
        students_data = json.loads(response)
        for s in students_data:
            student = Student(s["first_name"], s["last_name"], s["email"], s["pronouns"])
            student.allergies = s["allergies"]
            student.next_of_kin = s["next_of_kin"]
            school.students.append(student)
        print(f"Batch {i+1}/{batches} - {len(school.students)} students so far")
    except json.JSONDecodeError as e:
        print(f"Warning: Could not parse student batch {i+1}, skipping. Error: {e}")
```

### 5.3 Dropout Logic

```python
def run_dropout_logic(school):
    for student in school.students:
        if student.dropped_out:
            continue
        dropout_chance = 0.05
        if student.pronouns in ["he/him"]:
            dropout_chance += 0.02
        if random.random() < dropout_chance:
            student.dropped_out = True
            print(f"{student} has dropped out")
```

## 6. Key Python Patterns Learned Today

| Pattern | Notes |
|---------|-------|
| `__init__` and `self` | Constructor. `self` is like `this` in C++. All instance variables need `self.` |
| `__str__` | Like `operator<<` in C++. Controls what `print(object)` shows |
| Dunder methods | `__len__`, `__eq__`, `__repr__` — same concept as operator overloading |
| f-strings | `f"Hello {name}"` — variable interpolation inside strings |
| List comprehensions | `[x.strip() for x in items if x.strip()]` — filter and transform in one line |
| Tuple unpacking | `for threshold, letter in grades` — unpack tuple values into named variables |
| `try/except` | Wrap risky operations (JSON parsing, LLM calls) to handle failures gracefully |
| `from X import Y` | No `.py` extension. `from student import Student` brings the class in |
| venv | Virtual environment isolates dependencies. Always activate before working |
