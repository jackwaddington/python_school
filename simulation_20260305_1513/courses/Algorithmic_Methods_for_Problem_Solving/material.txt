 Algorithmic Methods for Problem Solving

1. **Divide and Conquer**
   - Definition: A problem-solving approach that breaks down a complex problem into smaller sub-problems, solves each sub-problem recursively, and combines their solutions to solve the original problem.
   - How it works: The process involves three main steps: divide (break down the problem), conquer (solve the sub-problems), and combine (combine the solutions of sub-problems to solve the original problem).
   - Concrete example: QuickSort algorithm, which sorts an array by first picking a 'pivot' element from the array, partitioning the other elements into two sub-arrays according to whether they are less than or greater than the pivot, and then recursively sorting the sub-arrays.

2. **Greedy Algorithms**
   - Definition: A problem-solving approach that makes the locally optimal choice at each stage with the hope of finding a global optimum. It selects the best immediate solution without looking ahead.
   - How it works: The algorithm makes the locally optimal choice in each step, hoping that this will lead to a globally optimal solution. However, it may not always find the optimal solution due to its short-sighted nature.
   - Concrete example: Knapsack problem where we have a set of items with weights and values, and a knapsack with a limited capacity. The goal is to choose a subset of items that maximize total value without exceeding the knapsack's capacity. A greedy algorithm would select items in order of decreasing value-to-weight ratio until the knapsack is full.

3. **Dynamic Programming**
   - Definition: An algorithmic technique used for optimization problems, where a problem is broken down into smaller overlapping sub-problems, solutions to these sub-problems are stored in a table (called memoization), and the solutions to the original problem are derived using the stored solutions.
   - How it works: The process involves two main steps: overlap (break down the problem into overlapping sub-problems) and store (memoize the solutions of sub-problems in a table).
   - Concrete example: Fibonacci sequence problem where we want to find the nth Fibonacci number. A dynamic programming solution involves constructing a table where the i-th cell stores the i-th Fibonacci number, and filling the table from bottom-up starting from the first two cells (0 and 1).

4. **Backtracking**
   - Definition: An algorithmic technique for finding all solutions to a problem recursively by generating candidate solutions incrementally, then eliminating dead ends through backtracking when no further progress can be made.
   - How it works: The algorithm starts with an initial solution and extends it in a systematic way, checking at each step if the current solution is still valid. If the current solution cannot be extended any further or if it violates some constraints, the algorithm backtracks to the previous step and explores a different branch.
   - Concrete example: The 8 Queens problem where we want to place 8 queens on an 8x8 chessboard such that no two queens threaten each other (i.e., they do not share the same row, column, or diagonal). A backtracking algorithm would start by placing a queen in the first row, then recursively place the remaining queens while ensuring they are safe from attack by previously placed queens. If no valid solution is found during this process, the algorithm backtracks to the previous step and tries a different placement for the current queen.
